Выполни коммит и пуш всех изменений. Все сообщения выводи на русском языке.

Шаги:
1. Выполни `git status` чтобы увидеть все изменения
2. Если изменений нет — сообщи "Нет изменений для коммита" и завершись
3. Выполни `git diff` и `git diff --staged` чтобы понять суть изменений
4. Добавь все изменённые и новые файлы в staging (`git add` с указанием конкретных файлов — НЕ используй `git add .` или `git add -A`, чтобы не добавить секреты или лишние файлы). Не добавляй файлы .env, credentials и прочие секретные файлы.
5. Создай коммит с коротким осмысленным сообщением на русском языке, описывающим суть изменений
6. Запуши в удалённый репозиторий (`git push`). Если upstream не настроен, используй `git push -u origin <текущая-ветка>`
7. Выведи итог: что было закоммичено и запушено
